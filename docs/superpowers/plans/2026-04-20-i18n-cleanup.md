# i18n Cleanup and Catalog Expansion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fully implement i18n across all frontend pages, removing hardcoded Chinese/English strings and expanding the frontend translation catalog to eliminate missing key warnings.

**Architecture:** 
1. Expand the `frontendCatalog` in `resources/js/lib/i18n.ts` to include all required keys.
2. Refactor components to use the `useTranslation` hook and `t()` function.
3. Standardize error message handling via i18n keys.

**Tech Stack:** React, Inertia.js, TypeScript, custom i18n library.
