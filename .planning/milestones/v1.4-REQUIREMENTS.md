# Requirements: Draupnir v1.4 Hardening & Refinement

**Defined:** 2026-04-12
**Core Value:** 提高系統穩定性、簡化維護流程、為未來功能擴展奠定基礎

## v1.4 Requirements

### 背景任務統一化 (Phase 18)

- [x] **JOBS-01**: 所有 cron/scheduled 任務使用統一的任務運行器（Bun.cron or similar）
- [x] **JOBS-02**: `ScheduleReportService` 從 `boot()` 遷出至統一任務管理器
- [x] **JOBS-03**: 移除 `boot()` 中的零散定時邏輯，集中定時任務定義
- [x] **JOBS-04**: 任務運行器支援單個任務重試和失敗恢復機制

### Alerts 模組解耦 (Phase 19)

- [x] **ALERTS-01**: 重構 Alerts 倉庫使用 ORM-agnostic `AggregateSpec` 模式（參考 Phase 17）
- [x] **ALERTS-02**: Alerts 模組不依賴任何核心業務邏輯層的實現細節
- [x] **ALERTS-03**: Alerts 系統可在無 DI 容器的情況下進行單元測試
- [x] **ALERTS-04**: Webhook 和 Email 適配器實現完全解耦
- [x] **ALERTS-05**: 警報配置、歷史記錄和發送日誌完全隔離

### CI 驗證加固 (Phase 20)

- [x] **CI-01**: Playwright E2E smoke tests（關鍵路徑：admin/member portal、CLI device flow、PDF 生成）集成進 CI pipeline
- [x] **CI-02**: 自動 TypeScript 類型檢查防止類型錯誤合併
- [x] **CI-03**: 測試覆蓋率驗證（最低 80%）於每次提交檢查
- [x] **CI-04**: 代碼品質檢查（linting、formatting）於 CI 中執行
- [x] **CI-05**: 提交歷史驗證防止無效提交信息進入 main
- [x] **CI-06**: DB migration schema drift 檢查（SQL migrations vs Drizzle schema 一致性）於 CI 執行
- [x] **CI-07**: Routes invariance 驗證（沿用 `scripts/routes-analyzer.ts` + `tests/Feature/routes-existence.test.ts`）於 CI 執行
- [x] **CI-08**: DI binding 完整性稽核（每個 port 都有 adapter 綁定、無漏綁）於 CI 執行

## Out of Scope

| 功能 | 原因 |
|------|------|
| Redis 持久化 | 後續版本考慮，目前 BullMQ 延期 |
| 運行時網關切換 | 網關是編譯時/ServiceProvider 決策 |
| 新的警報通知渠道 | v1.4 聚焦模組解耦，新渠道於 v2+ 實現 |
| 分佈式任務排隊 | 當前 Bun.cron 足夠，v2+ 考慮 BullMQ |
