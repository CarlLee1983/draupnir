# Requirements: Draupnir v1.3

**Defined:** 2026-04-12
**Core Value:** Proactive cost control — alerts, per-key attribution, and automated reports transform Draupnir from reactive observation to proactive cost management.

## v1.3 Requirements

Requirements for v1.3 milestone. Each maps to roadmap phases.

### Alerts

- [x] **ALRT-01**: User can configure cost threshold alerts (soft 80% / hard 100%) per organization
- [x] **ALRT-02**: User can configure cost threshold alerts per individual API key
- [x] **ALRT-03**: System sends email notification when cost threshold is breached
- [x] **ALRT-04**: System deduplicates alerts within a configurable window to prevent alert fatigue
- [x] **ALRT-05**: Alerts include severity levels (warning / critical) based on threshold tier
- [x] **ALRT-06**: User can register webhook endpoints for alert notifications (MVP)
- [x] **ALRT-07**: Webhook payloads are signed with HMAC-SHA256 for verification
- [x] **ALRT-08**: User can view alert history and delivery status

### Cost Breakdown

- [x] **COST-01**: User can view per-key cost breakdown for a given time period
- [x] **COST-02**: User can view per-key token usage efficiency metrics
- [x] **COST-03**: User can view per-model cost distribution across the organization
- [x] **COST-04**: User can view per-model aggregation with usage percentage

### Reports

- [x] **REPT-01**: Admin can configure scheduled PDF report delivery (weekly / monthly)
- [x] **REPT-02**: System generates server-side PDF reports with dashboard charts and cost data
- [x] **REPT-03**: System emails PDF reports to configured recipients on schedule
- [x] **REPT-04**: Report scheduling supports timezone selection to handle DST correctly

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| SMS/push notifications | High complexity, low ROI for B2B use case |
| ML anomaly detection | Excessive scope for v1.3; cost thresholds sufficient |
| Real-time WebSocket alerts | Post-sync evaluation (hourly) is acceptable latency |
| Multi-currency support | USD-only simplifies v1.3; revisit if needed |
| Hard enforcement (auto-disable keys) | Risk of disruption; alerts are informational only |
| Custom alert logic/rule builder | Premature complexity; fixed thresholds for v1.3 |
