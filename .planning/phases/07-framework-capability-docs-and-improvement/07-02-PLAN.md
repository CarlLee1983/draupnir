  <name>Task 1: Fix HandleCreditToppedUpService test GatewayError mock setup</name>
  <files>src/Modules/Credit/__tests__/HandleCreditToppedUpService.test.ts</files>
  <action>
Fix the test at approximately line 77-94. The problem is that `new (await import(...)).GatewayError(...)` is executed synchronously when passed to `failNext()`, but inside an `it()` async test. This causes a process-level throw.

Change from:
```typescript
it('Gateway 恢復失敗時應記錄失敗但不清除本地凍結狀態', async () => {
  ...
  mock.failNext(new (await import('@/Foundation/Infrastructure/Services/LLMGateway')).GatewayError(
    'Gateway error',
    'NETWORK',
    503,
    true,
  ))
```

To (defer the error construction):
```typescript
it('Gateway 恢復失敗時應記錄失敗但不清除本地凍結狀態', async () => {
  ...
  const GatewayErrorClass = (await import('@/Foundation/Infrastructure/Services/LLMGateway')).GatewayError
  const error = new GatewayErrorClass('Gateway error', 'NETWORK', 503, true)
  mock.failNext(error)
```

Or even simpler, move the import to the top of the file (not inside the test) if it doesn't break other patterns. The key is separating the import/construction from the `failNext` call to allow proper error deferral.

Complete the test following the existing pattern (assertions on result.failed, etc.) without changes to the logic.
