## Context

The specification document `docs/superpowers/plans/2025-05-15-english-only-api-responses.md` lists all modules requiring API message standardization:
- Auth (LoginRequest, RegisterRequest, RefreshTokenRequest, services, middleware, controller)
- Organization (request schemas, services)
- Contract (request schemas, services)
- AppModule (request schemas, services)
- AppApiKey (services)
- CliApi (services)
- Dashboard (services)
- DevPortal (services)
- Health (controller, services)
- SdkApi (controller, middleware, use cases)
- Credit (request schemas, services)

This plan focuses on the highest-volume and most user-visible modules: Auth, Organization, Contract, AppModule, and Credit. These modules have the most request validation and service-layer error messages.

## Pattern

**Request Validation (Zod schemas):**
```typescript
// BEFORE
email: z.string().email('不是有效的電子郵件')

// AFTER
email: z.string().email('Invalid email address')
```

**Service Error Fallback:**
```typescript
// BEFORE
message: error.message || '登錄失敗'

// AFTER
message: error.message || 'Login failed'
```

**Error Codes (NO CHANGE):**
```typescript
// Keep error codes unchanged
error: 'UNAUTHORIZED'  // NOT translated
```
