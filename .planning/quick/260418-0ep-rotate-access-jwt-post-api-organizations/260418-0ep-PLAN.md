<objective>
POST /api/organizations 成功後要 rotate access JWT：新 token 以 httpOnly cookie 寫回、
JSON 帶 `data.redirectTo`（`/manager/dashboard`），前端 modal 改讀 `redirectTo` 跳頁。
採方案 1A — 只在這一條路由掛 pendingCookies flush，不動其他 /api/* 路由、也不改
refresh token。

Purpose:
使用者剛建立組織、role 已從 member 升為 manager，但 JWT 仍是舊 role，
導致他會被導向 /member/* 頁面後遭 requireManager middleware 退回。rotate 即解此問題。

Output:
- Controller 注入 IJwtTokenService → 成功分支簽新 access token → setCookie + redirectTo
- Route 加 pendingCookiesMiddleware 讓 setCookie queue 真的 flush 到 Response
- Modal 前端改讀 redirectTo
- 新單元測試 + E2E journey 驗證完整流程
</objective>
